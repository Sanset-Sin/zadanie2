Интерактивный одностраничный дашборд на чистом JavaScript (ES6 Modules), собранный как практическое задание по UI-компонентам и ООП.

## Что внутри

- **ToDoWidget** — список задач с независимым состоянием
- **WeatherWidget** — погода по городу через Open-Meteo API + рекомендация на день
- **GitHubWidget** — профиль GitHub, публичная активность, тепловая карта и последние события
- **ServicePulseWidget** — демонстрационный график стабильности сервиса по часам
- **Dashboard** — управление коллекцией активных виджетов
- **UIComponent** — базовый абстрактный класс для всех компонентов

## Технологии

- HTML5
- CSS3
- JavaScript ES6+
- ES Modules
- OOP
- Fetch API

## Запуск локально

Открой проект через локальный сервер, например VS Code Live Server.
Просто двойным кликом по `index.html` запускать не рекомендуется, потому что используются ES6 modules.

## Публикация на GitHub Pages

1. Создать новый репозиторий на GitHub
2. Загрузить в него содержимое проекта
3. Открыть `Settings → Pages`
4. В разделе Build and deployment выбрать ветку `main` и папку `/root`
5. Сохранить настройки и дождаться публикации

Интерактивная панель управления UI-компонентами на чистом JavaScript (ES6 Modules) для практического задания №2.

## Что внутри

- **ToDoWidget** — локальный список задач
- **QuoteWidget** — локальные случайные цитаты
- **WeatherWidget** — получение текущей погоды через **Open-Meteo API**
- **GitHubWidget** — получение данных профиля через **GitHub REST API**
- **Dashboard** — управление коллекцией виджетов, добавлением и удалением компонентов
- **UIComponent** — базовый абстрактный класс для всех виджетов

## Структура проекта

```text
index.html
main.js
js/
  UIComponent.js
  Dashboard.js
  ToDoWidget.js
  QuoteWidget.js
  WeatherWidget.js
  GitHubWidget.js
styles/
  main.css
  components.css
README.md
```

## Как запустить локально

1. Открой папку проекта в VS Code / VSCodium.
2. Запусти локальный сервер:
   - через расширение **Live Server**, или
   - через `npx serve`, или
   - через любой другой простой HTTP server.
3. Открой `index.html` в браузере через локальный сервер.

> Важно: из-за ES6 modules лучше запускать через сервер, а не просто двойным кликом по файлу.

## Как выгрузить на GitHub Pages

1. Создай новый репозиторий на GitHub.
2. Загрузи в него **все файлы из этой папки** в корень репозитория.
3. Открой **Settings → Pages**.
4. В разделе **Build and deployment** выбери:
   - **Source:** Deploy from a branch
   - **Branch:** `main` (или `master`), папка `/root`
5. Сохрани настройки.
6. Через несколько минут GitHub Pages выдаст публичную ссылку на сайт.

## Что можно подставить от себя

- название проекта и текст в `index.html`
- стартовые виджеты в `main.js`
- цвета и стили в `styles/`
- собственный GitHub username для демонстрации
- свои цитаты или другую тематику
