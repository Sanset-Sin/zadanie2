# DevFlow Dashboard

Интерактивная панель управления UI-компонентами, выполненная на чистом JavaScript (ES6 Modules) с использованием принципов ООП.

## Что внутри

- **ToDo Widget** — независимый список задач с добавлением, удалением и отметкой выполнения.
- **Weather Widget** — загрузка текущей погоды по городу через Open-Meteo API + краткая рекомендация на день.
- **GitHub Widget** — профиль GitHub и heatmap публичной активности пользователя за последние 12 недель.
- **Service Monitor Widget** — демонстрационный аналитический виджет с графиком ошибок и задержки сервиса.

## Архитектура

- `main.js` — инициализация приложения и кнопок панели.
- `js/UIComponent.js` — базовый абстрактный класс виджета.
- `js/Dashboard.js` — управление коллекцией виджетов.
- `js/ToDoWidget.js` — виджет списка задач.
- `js/WeatherWidget.js` — виджет погоды через внешний API.
- `js/GitHubWidget.js` — GitHub-профиль и heatmap активности через внешний API.
- `js/ServiceStatusWidget.js` — виджет мониторинга сервиса.
- `styles/main.css` и `styles/components.css` — стили приложения и компонентов.

## Как запустить локально

Лучше запускать проект через локальный сервер, например:

- VS Code / VSCodium + **Live Server**
- `python -m http.server`

Не рекомендуется открывать `index.html` двойным кликом, потому что используются ES6-модули.

## Публикация на GitHub Pages

1. Создать новый репозиторий на GitHub.
2. Загрузить в него **всё содержимое папки проекта**.
3. Открыть `Settings → Pages`.
4. Выбрать публикацию из ветки `main` и корня `/root`.
5. Сохранить и дождаться ссылки на GitHub Pages.

## Примечание

Heatmap в GitHub Widget строится по **публичным событиям GitHub API**, а не по приватным contribution-данным профиля. Это позволяет показывать активность по любому открытому username без токена.
